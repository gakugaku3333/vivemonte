# 計画: EGS5相互検証パイプライン（scene.yamlを共通言語に）

作成日: 2026-07-13 / ステータス: **PHITSは導入せず、EGS5を相互検証の確立コードとして
継続使用することを正式決定（2026-07-17）。** 2026-07-15の方針転換（下記）で
主軸をPHITSからEGS5に切り替えた際は「PHITSライセンスが取得できれば追加の検証コード
として扱う」余地を残していたが、2026-07-17にユーザーの判断でPHITS導入自体を
取りやめ、EGS5を継続運用することを確定した。これに伴い、PHITS入力変換器
（旧Phase 3・4、下記）は実施しない。
同日以前の経緯: 2026-07-15、臨床寄り検証（BSF＋水ファントム線量分布）を新Phase 2
として追加し、変換器以降のフェーズを繰り下げ。2026-07-16、Phase 2b（PDD/側方
プロファイル）を実施し、事前登録基準に対する不合格（EGS5−ChatCarlo系統差−1.71%、
原因未確定）を検出。vive-auditor独立監査で数値・判定は再現確認済み（条件付き合格）。
詳細: [egs5_crosscheck/PDD_RESULTS.md](egs5_crosscheck/PDD_RESULTS.md)

**方針転換（2026-07-15、経緯）**: PHITSは国内でもライセンス申請から配布まで約3週間
かかることが判明し、抄録締切（2026-07-17 12:00）に間に合わないため、即日入手可能な
EGS5（KEK配布、登録不要、Stanford/SLACライセンス）にPhase 1相当を切り替えて実施した
（60 keV鉛筆ビーム＋水スラブ10cm、詳細・数値は[docs/egs5_crosscheck/RESULTS.md](egs5_crosscheck/RESULTS.md)）。
結果は良好（束縛コンプトン断面積で物理モデルを揃えた条件で相対0.88%・1.7σ一致）
だったため、今後の相互検証の主軸をPHITSからEGS5に変更した。

## 目的

ChatCarloの計算結果を確立コードEGS5と自動突き合わせできる体制を作り、
READMEの「教育・研究用」注記を実測比較データ付きで格上げする。
あわせて、以後の新機能（蛍光X線等）を追加するたびにEGS5と自動照合できる
継続的検証網を先に張る。

## 基本方針

「EGS5を万能に操るAI」は目指さない。**ChatCarloが表現できる範囲**
（box/sphere/cylinderの3プリミティブ＋X線管光子線源＋吸収線量/H*(10)タリー）
に限定した比較（自作ユーザーコード）を行う。

- 単一の真実源は人間が3Dプレビューで目視確認済みの scene.yaml。
  AIがEGS5入力を自由作文することは**しない**（MCコードの古典的危険＝
  文法的に通るが物理的に間違った入力が、もっともらしい数値を静かに返す
  ことへの対策。egs5-operatorエージェントに委任し、許容基準を実行前に固定する）
- ChatCarlo自身を独立推定量として使う。両コードの不一致が入力ミスの
  検出器になる（collision vs track-length 交差検証のコード間版）
- コード選択はEGS5。理由: ライセンス申請不要・即日入手可能（KEK配布）。
  ジオメトリ記述・T-Trackタリー相当の線量換算は自作ユーザーコードで書く必要があり
  PHITSほど機能は揃っていないが、即応性の利点が上回ると判断した

## フェーズ計画

### Phase 0 — 環境（唯一の人間依存ステップ、**完了・EGS5に読み替え**）

- [x] EGS5（KEK配布）の入手・macOSへのビルド確認（ライセンス申請不要、即日）

**完了条件**: サンプル入力がCLIから走り、出力ファイルが得られる。→ 満たした。

### Phase 1 — 手書き最小ケースで「地雷」を洗い出す（**完了・EGS5で実施**）

- [x] 単色鉛筆ビーム（60 keV）＋水スラブのEGS5ユーザーコードを**手書き**で作成
- [x] 一次透過率がBeer-Lambert則および `tests/test_transport.py` の
      ChatCarlo結果と一致することを確認（束縛コンプトン条件で相対0.88%・1.7σ一致）
- [x] つまずいた入力形式の罠を `docs/lessons_learned.md` に記録

**完了条件**: 3者（解析解・ChatCarlo・EGS5）が統計誤差内で一致した
比較表が残っている。→ 満たした（[RESULTS.md](egs5_crosscheck/RESULTS.md)）。

### Phase 2 — 臨床寄り検証: BSF＋水ファントム線量分布（EGS5、2026-07-15追加）

Phase 1の「一次透過率」は散乱を捨てた量なので、次は**散乱光子の輸送そのもの**を
臨床的に意味のある量で検証する。最終目標は胸部の入射表面線量（ESD）だが、
一足飛びに絶対ESDを比べるとmAs校正（SpekPy絶対フルエンス）という輸送と無関係な
差が混ざるため、「一度に1つの差だけを見る」原則（Phase 1のIBOUNDの教訓）に従い
2段構えにする。

**事前条件（物理モデル・比較条件を揃える — 実行前にここへ固定する）**:

- EGS5は**IBOUND=1**（束縛コンプトン）固定（Phase 1の教訓）
- ChatCarloはカーマ近似なので、EGS5側は高ECUT（電子を輸送せず局所吸収）または
  カーマ採点で揃える。電子平衡が崩れるビルドアップ域（表層サブmm）は比較から除外
- 採点グリッドは両コードで**ビン境界まで一致**させる（勾配の急な浅部で
  ビン中心ずれが見かけの差になるため）
- 空気側ボクセルは比較対象にしない（track-length推定量の境界効果、
  lessons_learned既載）— 採点はファントム内部のみ
- 許容基準は実行前に定義: 統計誤差1%未満のビンに限定して
  「2σかつ相対差2%以内」（Phase 1と同基準をビン単位に適用）
- 水・軟組織は低Zなので蛍光X線未実装の影響はほぼゼロ。鉛など高Zがらみの
  ケースは蛍光実装後の「寄与の定量実験」として温存し、ここでは扱わない

#### Phase 2a — 後方散乱係数（BSF）

- [ ] 幾何: 点線源＋矩形照射野＋水スラブファントム（例: 30×30×20 cm、SSD 100 cm）
- [ ] BSF＝（ファントム表面のカーマ、後方散乱込み）÷（同位置の自由空気カーマ）。
      **比なのでmAs校正・線源強度・立体角の差がすべてキャンセル**され、
      純粋に散乱輸送だけを比較できる
- [ ] 単色（60/80/100 keV）→ SpekPyスペクトル（同一の離散スペクトルを
      EGS5線源に渡す）の順に段階実施
- [ ] **三つ巴照合**: ChatCarlo・EGS5・公表MC計算値（Grosswendt、
      Petoussi-Henss et al. 1998 等。診断線質の水BSFは約1.3〜1.5）。
      文献アンカーがあるためコード間比較を超えた「現実」との照合になる

**完了条件**: 各条件のBSF比較表（ChatCarlo / EGS5 / 文献値、統計誤差付き）と
合否判定が `docs/egs5_crosscheck/` に残っている。

#### Phase 2b — 水ファントム線量分布（**完了 2026-07-16、ただし不合格判定を発見**）

- [x] 同一幾何で中心軸深部線量曲線（PDD）＋2深さ（表面近傍・10 cm）の
      側方プロファイルを比較
- [x] 正規化は **per-history絶対値（Gy/history）**。線源定義（同一スペクトル・
      照射野・SSD）さえ揃えればmAs校正なしで絶対スケールごと比較できる
- [x] ChatCarlo側は境界アーティファクト回避のためPhase 2aのtrack-length推定量
      （`--dose-grid`ではなく`_segment_box_overlap_cm`拡張）を使用（計画からの
      変更、理由はPDD_RESULTS.md参照）。EGS5側はegs5-operatorが47ビン分類の
      採点ユーザーコードを作成
- [x] 中心軸PDDは断面2×2 cmコラムで統計誤差1%未満を確保（ChatCarlo n=5e6,
      EGS5 n=1e8）
- [x] 形状は両コードで一致（不一致は位置に局在せず絶対値スケール全体に一様）

**完了条件**: 満たした。[PDD_RESULTS.md](egs5_crosscheck/PDD_RESULTS.md)に
ビン単位の合否表、[pdd60_comparison.png](egs5_crosscheck/pdd60_comparison.png)に
重ね描き図。**ただし結果は事前登録基準に対して不合格**（47ビン中37ビン不合格、
EGS5が一貫して−1.71%低い系統差）。vive-auditor独立監査で数値・判定は再現確認済み
（条件付き合格＝判定は信頼できるが原因分析は未確定）。原因は「ChatCarloのNIST
XAAMDI準拠μen/ρテーブルとEGS5のPEGS5内部断面積データが独立ソースである」ことが
有力仮説だが未検証。詳細・今後の対応はPDD_RESULTS.md参照。

#### 付記 — 胸部ESD/DRL照合（独立フェーズにしない）

Phase 2aのBSFに既存のmAs校正を掛ければ胸部ESDは追加計算ほぼなしで出る。
DRLs 2020（胸部ESD 0.3 mGy）との桁照合をRESULTSの1節として載せるにとどめ、
検証の主線（コード間比較）から正規化の差を切り離しておく。

### Phase 3 — 変換器と出力パーサ（**実施しない、2026-07-17決定**）

PHITS入力への機械変換器（`scene.yaml → PHITS入力`）は、PHITS導入自体を
取りやめたため実施しない。EGS5側の入力は各検証ケースごとに
egs5-operatorエージェントが自作ユーザーコードを書く運用（Phase 2a/2bの実績）を
継続する。scene.yamlの3プリミティブ・材料・SpekPyスペクトルはEGS5側でも
ChatCarloと同一ソース（xraylib/NIST）から手動で対応させる。

### Phase 4 — 相互検証ハーネス（**方針変更: 自動変換ではなくスキル化で対応**）

CLIによる自動変換ハーネス（`scene.yaml → PHITS入力 → PHITS実行`）は
Phase 3を実施しないため作らない。代わりに、Phase 5で確立した
「事前基準の固定→ChatCarlo実行→EGS5側をegs5-operatorに委任→3者比較→
vive-auditで監査」という半自動フローを標準運用とする。

- [ ] 許容基準を**事前に**文書化する運用を継続（例: 一次透過率は2σかつ2%以内、

      材料ごとに設定。ICRP/ICRU組成差で数%は正当にずれる —
      lessons_learned既載の教訓を反映）
- [ ] 代表シーン3種で実施: (1) 単純スラブ、(2) chest_room.yaml、
      (3) 鉛遮蔽強調シーン（後方散乱・重元素で差が出やすい条件）
- [ ] 結果をREADMEに反映（「教育・研究用」注記の書き換え、比較表の掲載）

**完了条件**: 3シーンの合否表がリポジトリに残り、READMEが更新されている。

## 想定される差異要因（不一致調査の初期チェックリスト）

| 要因 | 方向性 | 対処 |
|---|---|---|
| カーマ近似 vs 電子輸送 | ChatCarloが界面近傍で過小/過大 | 診断領域では電子飛程<1mm、ボクセルサイズ≫飛程なら無視可。差が出たらボクセル粗化して確認 |
| 蛍光X線の無視 | 重元素近傍でChatCarloが局所過大・遠方過小 | 既知の未実装項目。差が出れば実装優先度の根拠になる |
| 断面積ライブラリ差（EPDL版数等） | 全域で±1%程度 | 許容基準に織り込む |
| 組成規格差（ICRP vs ICRU-44） | 骨等で数% | 材料ごとの許容誤差（既存 `_LOOSE` 方式） |
| スペクトル離散化・線源の照射野一様近似 | 系統差 | 同一離散スペクトルを両者に渡し線源起因の差を排除 |

## リスクと中止条件

- EGS5とChatCarloの系統差が原因未確定のまま残る場合（Phase 2bの−1.71%系統差など）:
  crosscheckは「条件付き合格」として記録し、原因調査を別タスクとして継続する
  （合否判定自体は保留にしない）
- ChatCarloの表現力を超える要求（回転体、任意多面体等）を
  受けても**拡張しない**。scene.yamlスキーマの拡張が先

## やらないこと（スコープ外）

- PHITSの導入・入力変換器（2026-07-17、ユーザー判断で取りやめ。理由:
  ライセンス取得の待ち時間、EGS5で十分な精度の相互検証ができている実績）
- EGS5の汎用オペレーターAI化（`scene.yaml`全機能の自動EGS5変換）
- ChatCarlo側の物理追加（蛍光X線）— 本計画の**後**に、検証網の上で行う
