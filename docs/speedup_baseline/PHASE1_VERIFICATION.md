# Phase 1（断面積テーブル化）同等性検証結果

`docs/plan_transport_speedup.md` Phase 1完了時点の記録。設計判断4の(a)(b)(c)に対応。

## (a) 補間誤差（xraylib直接値との相対誤差）

`tests/test_xs_tables.py`（74件、全通過）で恒久的な回帰テストとして担保:
- 元素レベル（吸収端±0.5%を除く）: 実測worst caseは光電0.059%、コンプトン0.00008%、
  レイリー0.00004%（許容閾値0.2%に対し十分な余裕）
- 化合物レベル（同上、mu_rho vs xraylib \*_CP直接呼び出し）: 実測worst caseは
  lead_glassで0.002%
- 吸収端の鋭さ（オフセット20eV以上）: worst caseは0.065%（Pb K端近傍）

## 実装過程で見つかった落とし穴（テストで回帰防止済み）

1. **xraylib.EdgeEnergy()の公称値と実際のCS_Photoジャンプ位置は完全一致しない**
   （Al K端で最大0.038%相対、Zn L端で約0.2%）。固定の小さい探索窓では見つからず、
   隣接吸収端との間隔に応じた適応的な粗スキャン＋二分探索方式に変更して解決。
2. **亜鉛のL3-L2端間に近接閾値構造**（ジャンプ後に約1.8倍まで滑らかに上昇してから
   次の端に至る、物理的に実在する挙動）があり、基本格子密度だけでは最大約2%の
   誤差を生んだ。ジャンプ直後の追加補強点（`_NEAR_THRESHOLD_*`）で解消。

## (b) 全単体テスト

`pytest tests/ -q`: 223件全通過（既存149件＋新規`test_xs_tables.py`74件）。
実行時間もベースライン75.25秒→61.48秒に短縮（断面積呼び出し削減の副次効果、
テストスイート自体もxraylibを多用するため）。

## (c) EGS5相互検証代表ケースの再実行

Phase 1は3段階で実装した: (1)断面積のテーブル化（PCHIP）→(2)PCHIP評価コストが
xraylib削減分を相殺していたためnp.interpへ切替→(3)`_mu_and_parts_batch`で
μ計算と内訳計算の重複呼び出しを解消、`_element_cs_all`でphoto/compt/rayl
3種の検索(searchsorted)を1回にまとめ。以下は最終形(3)に対する再検証。

- `docs/egs5_crosscheck/run_chatcarlo_water60.py`（60 keV水スラブ、一次透過率）:
  出力が完全一致（12.589%、mu値も表示精度で同一）。水は吸収端から遠く、
  相互作用回数も少ないため差が出にくいケース。
- `docs/egs5_crosscheck/fluorescence_lead_150kev/`（150 keV鉛スラブ、K殻蛍光、
  Pb K端88 keV直上を主に扱う最も検証価値の高いケース、n=1,000,000）:
  `chatcarlo_results.json`（既存の正典値）と比較すると、**単純な値の一致では
  なく統計的同等性の確認が必要だと判明した**——n_escaped等の生カウントは
  蛍光ON/OFFいずれも0.2〜0.4%ずれた（完全一致ではない）。原因を特定した:
  無散乱透過（round 1のみで決まる、`uncollided_transmission`）は前後で
  **完全一致**（diff 0σ）だったのに対し、複数回散乱を経た結果を含む
  `transmitted_energy_fraction`は前後でズレる。輸送カーネルは1本の共有RNG
  ストリームを使い、各ラウンドで`rng.random(len(alive))`と生存数ぶんだけ
  乱数を消費するため、ごく僅かな補間誤差でどこか1件でも相互作用種別の
  判定が変われば、その後の生存数が変わり、以降全ラウンドの乱数消費位置が
  ズレて無関係な光子群にも連鎖する（決定論的カオス的感度、single-stream
  vectorized MCに内在する性質でバグではない）。
  そのため設計判断4(b)の通り統計的同等性で判定した:
  transmitted_energy_fraction の前後差は0.78σ（OFF）・0.42σ（ON）、
  主要指標のΔ（蛍光による透過エネルギー変化量）は前後で相対0.47%の差
  （0.062193→0.062488）——いずれも統計誤差の範囲内で、Phase 4の事前登録
  合否判定（Δ比0.7〜1.3、実測0.9919）を揺るがす差ではない。

**結論**: 遮蔽計算で最も重要なPb K端周辺を含め、物理結果は統計的に保存されている。
「同一seedで生カウントが完全一致するか」ではなく「物理量が統計誤差内で一致するか」
で判定する必要があると実例で確認できた（設計判断4の想定通り）。Phase 1は合格。
