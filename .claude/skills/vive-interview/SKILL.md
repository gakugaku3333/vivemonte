# vive-interview — シミュレーション要件のヒアリング

このスキルは**聞き取りと要件の確定だけ**を担う。いきなりscene.yamlを書き始めず、
以下の段階で質問して意思を確認してから草案を作る。ユーザーが既に伝えた情報は
再質問しない（聞くのは不明な項目だけ）。質問はAskUserQuestionを使い、
1回につき1〜3項目まで。選択肢には診断領域の代表値を既定案として示す。

## 段階1: 目的の確認（最初に必ず聞く）

何を評価したいのかで、その後の質問・グリッド設計・報告する量が変わる:

- **患者被ばく**（入射表面線量・臓器近傍の吸収線量）→ 患者体表に細かいグリッド
- **操作者・第三者の被ばく**（防護衣の内外、操作室内のH*(10)）→ 評価点位置に細かいグリッド
- **遮蔽評価**（壁・鉛遮蔽の透過後線量）→ 遮蔽の背後に評価点
- **室内の空間線量分布の全体像** → 部屋全体の粗いグリッド

注意: 「グリッド全体の最大値」は空気ボクセルに落ちて非物理的になりやすい
（docs/lessons_learned.md）。目的を聞くのは、**実在する評価位置**を最初に
特定するためでもある。

## 段階2: 撮影条件

不明な項目のみ質問する。部位・検査種別が分かれば代表値を既定案として提示:

| 項目 | scene.yamlキー | 代表値の例 |
|---|---|---|
| 管電圧 | `source.kvp` | 胸部120、腹部80、四肢60 kV |
| 管電流時間積 | `source.mas` | 胸部2〜5 mAs（指定すると絶対値[Gy]/[pSv]に校正、未指定なら相対値のみ） |
| CTDIvol | `source.ctdi_vol_mGy` | CTはこちらを推奨: コンソール表示値をそのまま入れる（`ctdi_phantom: body/head`も確認）。masと同時指定は不可 |
| 総濾過 | `source.filtration_mm_al` | 2.5 mmAl（法令下限） |
| 照射野・SID | `source.field.size_cm` / `sid_cm` | 半切35×43 cm、SID 180 cm（胸部立位） |
| 円錐照射野 | `source.field.shape: cone` + `diameter_cm` | CBCT・広角視野はこちら（立体角一様、回転と合成可） |
| ヒール効果 | `source.heel_effect: true` + `anode_direction` | 陽極・陰極の向きは段階3（ジオメトリー）で毎回必ず確認する（下記） |

mAsは「実測相当の絶対線量が欲しいか」で要否が決まるので、段階1の目的と
セットで確認する。

## 段階3: ジオメトリー

座標系はcm、z軸鉛直上向き、床上面z=0。**検査種別が分かったら、まず
examples/ のベーステンプレートから最も近いものを雛形に選ぶ**（ゼロから
書かない）。

**手順**: `ls examples/*.yaml` で現在のテンプレート一覧を確認し、各ファイル
冒頭のコメント（検査種別・想定シナリオ・「カスタマイズの起点」）を読んで
最も近いものを選ぶ。下表はこのSKILL.md作成時点の一覧なので参考程度にし、
`ls`結果と食い違っていたら`ls`とファイル本体を正とする（テンプレート追加時に
この表の更新を忘れても事故らないようにするため）:

| 検査種別 | テンプレート | 想定シナリオ |
|---|---|---|
| 一般撮影・立位胸部 | `examples/chest_room.yaml` | 撮影室の空間線量・遮蔽 |
| 一般撮影・臥位（腹部等） | `examples/abdomen_table.yaml` | テーブルBucky、天井管球 |
| CT | `examples/ct_room.yaml` | 連続回転＋ヘリカル位相平均近似（scan_length_cm。⚠️ボウタイ・mA変調未対応） |
| アンギオ/IVR・透視 | `examples/angio_room.yaml` | アンダーテーブル管球、術者＋防護衣＋天吊り遮蔽つき |
| ポータブル（病室） | `examples/portable_ward.yaml` | 2m隣の患者への散乱線（第三者被ばく） |
| 単色/平行ビーム（物理検証用） | `examples/water_phantom_pdd_ocr.yaml` | 相互検証（vive-crosscheck）でPDD/OCRを比較する際の雛形 |

テンプレートにない検査種別（マンモグラフィ、歯科等）は最も近いものを
コピーして改変する。

雛形を選んだ上で、以下を確認:

1. **患者**（いる場合）: 体位（立位/臥位）、撮影部位の高さ・向き
2. **部屋**: 壁・床の材質（コンクリート厚など）。省略可（開放空間=airでも計算は
   回る）だが、散乱線評価なら壁は入れるべきと添える
3. **遮蔽・器物**: 鉛遮蔽、検出器パネル、防護衣など。材質は
   `chatcarlo/materials.py` の登録名で指定（water, soft_tissue, bone, aluminum,
   lead, concrete等。未登録材質を求められたら登録可否を先に確認）
4. **線源の位置・方向**: 患者/評価点との位置関係を言葉で復唱して確認する
   （「患者の前方180cmから水平に+y方向へ照射、で合っていますか？」）
5. **陽極・陰極の向き（ヒール効果、毎回必ず確認する）**: 線源の位置・方向が
   決まったら、AskUserQuestionで次を聞く（省略しない — 忘れると静止撮影
   （一般撮影・アンギオ/透視・ポータブル）で頭尾方向の線量非対称を
   見落とす。CTは回転で平均化されるため優先度は下がるが同様に聞く）:
   - 陽極方向を把握しているか（はい→向きを指定 / いいえ・不明→
     `heel_effect`なしで進める、代表値も提示しない — 実機の管球仕様書
     依存の情報を当て推量しない）
   - 「はい」なら、線源の方向（中心軸）に対してどちら向きが陽極側かを
     具体的な向き（例: 「頭側が陰極、足側が陽極」「+x側が陽極」）で聞き、
     `source.anode_direction`（ビーム中心軸と直交する成分が必要）に変換
     して復唱確認する
   - 有効化したら`source.heel_effect: true`をscene.yamlに設定する

## 段階4: 計算条件と最終確認

1. history数と解像度の案を提示（既定: `-n 1e6 --resolution 2`。目安:
   3e5 historiesで約15秒、1e7超は時間予告）
2. **確定した要件の一覧を表にして復唱**し、AskUserQuestionで最終承認を取る
3. 承認後、`mkdir -p runs/<シーン名>` してから、復唱した要件表をそのまま
   `runs/<シーン名>/requirements.md` に書き出す（`examples/chest_room.yaml`
   なら `runs/chest_room/requirements.md`。この `runs/<シーン名>/` は
   vive-checkが書く成果物一式の置き場所と共通で、`.gitignore`済みの作業用
   ディレクトリ）。これは監査官がscene.yamlを**物理的な正しさ**だけでなく
   **ユーザーの意図との一致**まで検証できるようにするための一次記録であり、
   値を代表値で埋めた項目には「代表値（未確認）」と明記する（黙って確定値の
   ように書かない）
4. scene.yamlを書き、`chatcarlo validate` を通す
5. そのままvive-checkスキル（関門1: ジオメトリー確認）に引き継ぐ。
   その際 `runs/<シーン名>/requirements.md` のパスも申し送る

## 聞き取りの原則

- 不明瞭な指示（「だいたいでいい」「普通の部屋」等）はそのまま解釈せず、
  自分の解釈を具体値にして「この理解で合っていますか」と確認する
- ユーザーが答えられない項目は代表値を提案して「この値で進めます」と明示する
  （黙って仮定しない）
- 物理的に疑わしい要望（焦点が物体内部、濾過なし等）はvalidateの警告を待たず
  ヒアリング段階で指摘する
